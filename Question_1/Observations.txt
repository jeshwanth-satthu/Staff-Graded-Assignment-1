1. Command: id

Observation:

The 'id' command was executed to verify the identity and privilege level of the currently logged-in user. This command displays the User ID (UID), Group ID (GID), and all groups associated with the user account. The output obtained was:

uid=1000(jeshw) gid=1000(jeshw) groups=1000(jeshw),4(adm),24(cdrom),27(sudo),30(dip),46(plugdev),100(users)

From the output, it is confirmed that the current user is 'jeshw'. The user belongs to several Linux groups including adm, cdrom, sudo, dip, plugdev, and users. Membership in the 'sudo' group indicates that the user has administrator privileges and can perform system administration tasks when required. Verifying user identity is an important first step before accessing or managing system resources.

2. Command: echo $SHELL

Observation:

The 'echo $SHELL' command was used to determine the default shell currently assigned to the logged-in user. The command returned the following output:

/bin/bash

This result confirms that the Bash (Bourne Again Shell) is the active command-line shell. Bash is the default shell in most Linux distributions, including Ubuntu, and provides an interface for executing Linux commands, writing shell scripts, automating repetitive tasks, and managing the operating system. Confirming the active shell ensures that the user is working in the expected command-line environment.

Observation:

The 'pwd' (Print Working Directory) command displays the complete path of the directory in which the user is currently working. The output obtained was:

This confirms that all Question 1 activities are being performed inside the designated assignment directory. Verifying the current working directory helps prevent files from being created in the wrong location and ensures that the assignment remains properly organized according to the required folder structure.

The command is useful for verifying the contents and properties of files in a Linux directory.

5. Command: ping -c 4 google.com

Observation:

The 'ping -c 4 google.com' command was used to verify internet connectivity by sending four ICMP Echo Request packets to Google's server. The command produced the following result:

4 packets transmitted
4 packets received
0% packet loss

The successful transmission and reception of all four packets confirms that the Linux system has an active and stable internet connection. The response times displayed in milliseconds indicate that communication with the remote server was successful without any packet loss. This verifies that the system is properly connected to the network and is capable of accessing online resources required for Linux administration tasks.
